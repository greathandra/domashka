JSON-LD

Рекомендуется Google Данные Schema.org в блоке script — без изменения видимого HTML.

Преимущества

  • Чистое разделение данных и вёрстки
  • Легко генерировать на сервере или в CMS
  • Удобно для SPA — можно добавлять динамически
  • Google явно рекомендует этот формат

Базовая структура

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebPage",
  "name": "Название страницы",
  "description": "Описание страницы",
  "url": "https://example.com/page"
}
</script>

Пример: товар (Product)

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Беспроводные наушники Pro",
  "image": "https://example.com/headphones.jpg",
  "description": "Шумоподавление и 30 часов автономности",
  "brand": { "@type": "Brand", "name": "AudioTech" },
  "offers": {
    "@type": "Offer",
    "price": "8990",
    "priceCurrency": "RUB",
    "availability": "https://schema.org/InStock"
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.8",
    "reviewCount": "127"
  }
}
</script>

Пример: FAQ

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "Что такое JSON-LD?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "Формат записи структурированных данных в JSON."
    }
  }]
}
</script>
На этой странице в <head> уже есть JSON-LD блок типа TechArticle — посмотрите исходный код.
